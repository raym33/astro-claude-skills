# Contributing

Thanks for contributing to Astro Claude Skills.

## Scope

This repository is for generic, reusable Claude Code skills for premium Astro site work.

Please keep contributions:

- generic
- brand-agnostic
- client-agnostic
- focused on structure, workflow, and reusable section systems

Do not add:

- client names
- source site brand names
- copied brand identities
- reference URLs from mirrored or imported websites

## What to contribute

- better skill instructions
- cleaner workflow references
- stronger capsule selection logic
- improved page recipes
- missing delivery guardrails
- generic examples that do not expose source identities

## Style

- Write everything in English.
- Keep instructions concise and operational.
- Prefer small, composable reference files over one large skill file.
- Optimize for Claude Code usefulness, not for human marketing copy.

## Skill design rules

- Use lowercase hyphenated names.
- Keep `SKILL.md` focused.
- Move detailed guidance into `references/`.
- Use generic terms like `hero`, `case-list`, `chapter-nav`, or `pricing-comparison`.
- If a pattern came from a real-world source, abstract it before adding it here.

## Validation

Before opening a PR, check:

1. No client or source brand names appear in the new files.
2. The instructions still make sense without private project context.
3. The skill is reusable across multiple Astro projects.
4. File paths and cross-references are valid.

## Pull requests

Good pull requests are:

- narrow in scope
- easy to review
- explicit about what changed
- careful about keeping the repository generic
