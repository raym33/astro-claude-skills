# Capsule catalog

This file exists to help Claude choose stronger sections faster.

## Common capsules

- `hero`
  Opening statement, premium introduction, main proposition
- `feature-pair`
  Alternating media and copy rows
- `editorial-quote`
  Signature quote or authored endorsement
- `join-cta`
  Final conversion block
- `logo-cloud`
  Partners, clients, ecosystem trust
- `product-tabs`
  Segmented explanation for services or product areas
- `proof-strip`
  Compact trust interruption
- `resource-cards`
  Services, units, capabilities, methods
- `sticky-steps`
  Process and methodology
- `testimonial-stories`
  Longer human proof
- `faq-list`
  Visible answer engine with strong SEO utility
- `service-map`
  Equal-weight service overview grid

## Selection rules

- Reuse an existing capsule if it preserves the source honestly.
- Build a site-only component when forcing a capsule would damage the original structure.
- Port a missing capsule when:
  - it is central to the site
  - it is likely to repeat
  - a weak substitute would reduce quality

## Principle

The inventory can stay relatively small and still produce premium sites if the patterns are structural and flexible.
